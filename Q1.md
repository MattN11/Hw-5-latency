# Latency Numbers

| Item | Jeff Dean 2009 | Modern Value | Improved? |
|---|---:|---:|---|
| L1 cache reference | 0.5 ns | about **0.9 ns** | Not really |
| Branch mispredict | 5 ns | about **3.4–3.6 ns** | Yes |
| L2 cache reference | 7 ns | about **2.7 ns** | Yes, a lot |
| Mutex lock/unlock | 25 ns | about **20–40 ns** | Not much |
| Main memory reference | 100 ns | about **85 ns** | Slightly |
| Read 1 MB sequentially from memory | 250,000 ns | about **24,000–26,000 ns** | Yes, a lot |
| Round trip within same datacenter | 500,000 ns | about **30,000 ns** | Yes, a lot |
| Disk seek | 10,000,000 ns | about **7,500,000 ns** | Slightly |
| Read 1 MB sequentially from disk | 20,000,000 ns | about **3,500,000–3,900,000 ns** | Yes, a lot |
| Send packet CA->Netherlands->CA | 150,000,000 ns | about **132,000,000–149,000,000 ns** | Not much |

## Prompt Used

I looked at Jeff Dean’s 2009 latency numbers and compared them to modern numbers for cache, memory, disk, and network operations. Then I checked whether each one has gotten faster or stayed mostly the same using newer sources.
