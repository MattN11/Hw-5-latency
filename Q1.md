# Latency Numbers

| Item | Jeff Dean 2009 | Representative modern value | Improved? | Notes / source |
|---|---:|---:|---|---|
| L1 cache reference | 0.5 ns | about **0.9 ns** | Not really | Zen 3 L1 latency is 4 cycles at 4.45 GHz, about 0.90 ns. Source: 7-cpu.com |
| Branch mispredict | 5 ns | about **3.4-3.6 ns** | Yes | Zen 3 branch misprediction penalty is 15-16 cycles at 4.45 GHz. Source: 7-cpu.com |
| L2 cache reference | 7 ns | about **2.7 ns** | Yes, a lot | Zen 3 L2 latency is 12 cycles at 4.45 GHz. Source: 7-cpu.com |
| Mutex lock/unlock | 25 ns | roughly **20-40 ns** | Not clearly | Depends on implementation and contention. Estimate based on atomic operation costs. Source: Agner Fog instruction tables |
| Main memory reference | 100 ns | about **85 ns** | Slightly | Zen 3 RAM latency is about 84.6 ns. Source: 7-cpu.com |
| Read 1 MB sequentially from memory | 250,000 ns | about **24,000-26,000 ns** | Yes, about 10x | Based on about 41 GB/s RAM read bandwidth. Source: 7-cpu.com |
| Round trip within same datacenter | 500,000 ns | about **30,000 ns** | Yes, a lot | Modern datacenter RTT is often in the tens of microseconds. Source: CMU paper |
| Disk seek | 10,000,000 ns | about **7,500,000 ns** | Slightly | Modern enterprise HDD average read seek time. Source: Toshiba MG09 spec |
| Read 1 MB sequentially from disk | 20,000,000 ns | about **3,500,000-3,900,000 ns** | Yes, a lot | Based on modern enterprise HDD transfer rates. Source: Toshiba MG09 spec |
| Send packet CA->Netherlands->CA | 150,000,000 ns | about **132,000,000-149,000,000 ns** | Not much | Still limited mostly by distance and routing. Source: WonderNetwork ping data |
